CS2-DMA External ESP Tool

A free, read-only external CS2 tool built with C++, using DMA hardware via FPGA devices. ESP, Web Radar, Grenade Helper — no data upload, no telemetry, no server connection. Actively maintained with continuous updates and bug fixes.

Features

Everything You Need

Full-featured external ESP system with DMA hardware optimization, designed for minimal latency and maximum stability.

👁

Visuals / ESP

Box, Bone, Health Bar, Armor Bar, Weapon, Distance, Player Name, Eye Ray, Snaplines, Head Dot, Safe Zone — fully customizable.

💣

Bomb ESP

Planted / Carried / Dropped / Defusing status display with explosion and defuse countdown timers.

💥

Projectile ESP

Real-time display of in-flight flashbangs, smokes, HE grenades, molotovs, and decoys with explosion radius rendering.

📡

Web Radar

Built-in WebSocket server. Any device on the LAN can view real-time radar via browser.

🎯

Grenade Helper

Per-map preset throw positions with real-time direction arrow and distance indicator. Record, edit, and delete custom positions.

⚡

DMA Optimization

Scatter batch reads, on-demand field reading, tiered entity caching, zero-copy snapshot — sub-frame latency.

🛠

Config System

Create / save / load / delete multiple configs. Auto-loads _autosave.config on startup. Full JSON-based settings.

🛡

Crash Handler

SEH + std::terminate capture. Auto-generates .log + .dmp with recent logs, feature state, and system info.

Architecture

Multi-Thread + Snapshot

Clean separation of concerns with lock-free rendering and sub-frame data latency.

C++17
Language Standard
6
Concurrent Threads
<1
Frame Data Latency
80%+
Redundant Read Reduction
ConnectionThread  —— Game process lifecycle management (state machine)
DataThread        —— Core pipeline: matrix → local player → entities → scatter read
SlowUpdateThread  —— Low-frequency: entity list base address, map name
KeysCheckThread   —— Keyboard state polling (DMA reads kernel key state)
WebRadarThread    —— WebSocket broadcast: GameSnapshot → JSON
Main Thread       —— ImGui window + ESP rendering (read-only Snapshot)

Data flow:
  DataThread —— scatter DMA read —→ GameSnapshot (shared_mutex)
      ├—→ Render Thread  (shared_lock, read-only)
      └—→ WebRadar Thread (shared_lock, read-only)
Quick Start

Get Started in Minutes

Download the release package and start using immediately — no build required.

1

Download the Latest Release

Go to the Releases page and download CS2-DMA-Release.zip.

2

Connect FPGA Device

Ensure your DMA hardware is properly connected to the secondary machine.

3

Launch CS2 on the Primary Machine

Open Counter-Strike 2 and join a match on the gaming machine.

4

Run cs2.exe on the Secondary Machine

The program automatically initializes DMA, searches for the game process, and starts rendering ESP.

5

Press F8 to Open Menu

Toggle features on/off. Configure Visuals, Radar, Grenade Helper, Settings, and Config profiles.

# Build from source (optional) git clone https://github.com/chao-shushu/CS2-DMA.git cd CS2-DMA & "C:\Program Files\Microsoft Visual Studio\18\Community\MSBuild\Current\Bin\MSBuild.exe" ` "dma.slnx" /p:Configuration=Release /p:Platform=x64 /t:Rebuild /m
